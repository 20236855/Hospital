### 1. 必需软件

- Maven 3.6+
- MySQL 5.7+
- Redis 5.0+
- Nacos 2.0+

| 服务名称 | 端口 | 说明 |

| Nacos | 8848 | 服务注册与配置中心 |
| Redis | 6379 | 缓存服务 |
| MySQL | 3306 | 数据库服务 |
| ruoyi-gateway | 8080 | 网关服务 |
| ruoyi-auth | 9200 | 认证服务 |
| ruoyi-system | 9201 | 系统服务 |

| his-patient | 9210 | 患者服务 |
| his-register | 9212 | 挂号服务 |
| his-emr | 9213 | 电子病历服务 |

## 三、后端服务启动

### 1. 启动顺序建议

**第一步：启动基础设施服务**
- 启动 MySQL 服务
- 启动 Redis 服务
- 启动 Nacos 服务

**第二步：启动核心服务**
- 启动 ruoyi-auth（9200）
- 启动 ruoyi-system（9201）
- 启动 ruoyi-gateway（8080）

**第三步：启动业务服务**
- 启动 his-doctor（9211）
- 启动 his-patient（9210）
- 启动 his-register（9212）
- 启动 his-emr（9213）

### 2. 启动方法

#### 方法一：使用 IDE 启动（推荐开发环境）
1. 使用 IntelliJ IDEA 或 Eclipse 打开 RuoYi-Cloud 项目
2. 等待 Maven 依赖下载完成
3. 找到各服务的主启动类，右键点击 "Run" 启动：
   - `ruoyi-auth/src/main/java/com/ruoyi/auth/RuoYiAuthApplication.java`
   - `ruoyi-modules/ruoyi-system/src/main/java/com/ruoyi/system/RuoYiSystemApplication.java`
   - `ruoyi-gateway/src/main/java/com/ruoyi/gateway/RuoYiGatewayApplication.java`
   - `ruoyi-modules/his-doctor/src/main/java/com/ruoyi/hisdoctor/HisDoctorApplication.java`
   - `ruoyi-modules/his-patient/src/main/java/com/ruoyi/patient/HisPatientApplication.java`
   - `ruoyi-modules/his-register/src/main/java/com/ruoyi/register/HisRegisterApplication.java`
   - `ruoyi-modules/his-emr/src/main/java/com/ruoyi/emr/HisEmrApplication.java`

#### 方法二：使用 Maven 命令启动
在各服务目录下执行：
```bash
cd ruoyi-auth
mvn clean install
mvn spring-boot:run
```

#### 方法三：打包后启动
```bash
# 打包
cd RuoYi-Cloud
mvn clean package -DskipTests

# 启动各服务
java -jar ruoyi-auth/target/ruoyi-auth.jar
java -jar ruoyi-modules/ruoyi-system/target/ruoyi-system.jar
java -jar ruoyi-gateway/target/ruoyi-gateway.jar
java -jar ruoyi-modules/his-doctor/target/his-doctor.jar
java -jar ruoyi-modules/his-patient/target/his-patient.jar
java -jar ruoyi-modules/his-register/target/his-register.jar
java -jar ruoyi-modules/his-emr/target/his-emr.jar
```

### 3. 服务启动验证
访问 Nacos 控制台：http://127.0.0.1:8848/nacos
- 用户名：nacos
- 密码：nacos

在 "服务管理" -> "服务列表" 中确认所有服务都已注册成功。

## 六、常见问题

### 1. Nacos 连接失败
- 检查 Nacos 是否启动
- 确认 Nacos 地址配置正确（127.0.0.1:8848）

### 2. 数据库连接失败
- 检查 MySQL 是否启动
- 确认数据库连接信息正确
- 确认数据库 ry-cloud 已创建

### 3. Redis 连接失败
- 检查 Redis 是否启动
- 确认 Redis 地址配置正确（127.0.0.1:6379）
- 如果 Redis 未启动，健康检查已禁用，不影响服务启动

### 4. 前端页面报错 404
- 确认所有后端服务已启动
- 确认网关服务已启动
- 检查 Nacos 中服务是否注册成功

### 5. 登录失败
- 确认用户是否在数据库中存在
- 确认验证码输入正确
- 检查后端日志查看具体错误
